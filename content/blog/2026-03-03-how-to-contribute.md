Every part of Bahrain.js is open source — this website, our packages, our event listings. They are all files in a Git repo, and every one of them accepts pull requests. You do not need permission to start. You just need a GitHub account.

## Start Small

The fastest way to become a contributor is to fix something small. Here are three things you can do right now.

### Fix a Typo or Improve a Description

Every page on this site is a Markdown file inside the `content/` directory.

1. Open the [Bahrain.js GitHub repo](https://github.com/bahrain-js/bahrain.js)
2. Navigate to the file you want to edit under `content/`
3. Click the pencil icon and make your change
4. Submit a pull request

That single PR makes you a contributor. Your name appears on the [People](/people) page automatically.

### Write a Blog Post

Share something you built, learned, or discovered. Community blog posts are one of the highest-value contributions because they help other developers in Bahrain.

1. Create a new `.md` file in `content/blog/` using the format `YYYY-MM-DD-your-title.md`
2. Add the required frontmatter: `title`, `date`, `author`, `author_github`, `description`, and `tags`
3. Write your content in Markdown
4. Submit a pull request

Not sure what to write about? Here are some ideas:
- A project you built and what you learned
- A JavaScript concept explained in your own words
- A comparison of tools or frameworks you have used
- Tips from your job search or interview experience in Bahrain

### Add an Event to the Calendar

Running a meetup, workshop, or coding session? Add it so the community can find it.

1. Go to the [Submit Event](/events/submit) page
2. Fill in the details — title, date, format, description
3. Submit and a team member will review it

## Level Up: Code Contributions

When you are ready for something more substantial, here is how to find meaningful work.

### Find a Good First Issue

Browse the repos at [github.com/bahrain-js](https://github.com/bahrain-js) and filter for the `good-first-issue` label. These are scoped, well-documented tasks designed for new contributors.

### The Path to Publishing

Contributing to Bahrain.js follows a natural progression:

1. **Share an idea** — Bring it to a meetup or post it in the community chat
2. **Build it together** — Work on it during a builder session with other members
3. **Ship it** — Publish the finished package under `@bahrain.js` on npm

Every package we publish is a community achievement. Your name stays on it.

## Recognition

We believe contributions speak louder than follower counts. Every contributor — whether you fixed a typo or shipped a package — is visible on the [People](/people) page. We track what you build, not how many people follow you.
