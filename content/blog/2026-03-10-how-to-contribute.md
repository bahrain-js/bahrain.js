# How to Contribute to Bahrain.js

Everything at Bahrain.js is open source. This website, our packages, even our event listings — they're all just files in a Git repo. Here's how to get involved.

## The easy wins

### Fix a typo or improve content

Every page on this website is a Markdown file. Found a typo? Want to improve a description?

1. Click the link to our GitHub repo
2. Find the file under `content/`
3. Click "Edit" and submit a PR

That's it. You just became a contributor.

### Add a blog post

Write a post about something you've built, learned, or want to share with the community.

1. Create a new `.md` file in `content/blog/`
2. Add frontmatter (title, date, author, description, tags)
3. Write your content in Markdown
4. Submit a PR

### Add an event

Running a meetup or session? Add it to the calendar.

1. Create a new `.md` file in `content/events/`
2. Fill in the frontmatter (title, date, format, type)
3. Add the event description
4. Submit a PR

## Level up: code contributions

### Find a good first issue

Check our repos at [github.com/bahrain-js](https://github.com/bahrain-js) and look for issues labeled `good-first-issue`.

### The open source pipeline

1. **Got an idea?** Share it in the community chat or at a meetup
2. **Build it** at a builder session
3. **Ship it** — publish under `@bahrain.js` on npm

## Recognition

Every contributor is visible on the [People](/people) page. We track contributions, not follower counts.
